API Endpoints

Complete reference for all API endpoints.

Base URL​

All endpoints are prefixed with the base URL. The API runs on an automatically selected port (default: 3001-3010).

Example: http://localhost:3001/api

Health Check​

GET /health​

Check API server status.

Response:

{
  "status": "ok",
  "timestamp": "2024-01-01T12:00:00.000Z"
}

Projects​

GET /api/projects​

List all projects.

Response: Project[]

Example:

curl http://localhost:3001/api/projects

GET /api/projects/tags​

Get all unique tags across all projects.

Response: string[]

Example:

curl http://localhost:3001/api/projects/tags

GET /api/projects/:id​

Get project details by ID.

Parameters:

  • id (path): Project ID (numeric)

Response: Project

Example:

curl http://localhost:3001/api/projects/1

POST /api/projects​

Add a new project.

Request Body:

{
  "name": "My Project",
  "path": "/path/to/project"
}

Response: Project (201 Created)

Example:

curl -X POST http://localhost:3001/api/projects \
  -H "Content-Type: application/json" \
  -d '{"name": "My Project", "path": "/path/to/project"}'

Error Responses:

  • 400 - Missing name or path, path doesn't exist, or path is not a directory
  • 409 - Project with this path already exists

PUT /api/projects/:id​

Update project.

Parameters:

  • id (path): Project ID (numeric)

Request Body:

{
  "name": "Updated Name",
  "description": "Project description",
  "framework": "react",
  "tags": ["frontend", "react"]
}

All fields are optional. Only provided fields will be updated.

Response: Project

Example:

curl -X PUT http://localhost:3001/api/projects/1 \
  -H "Content-Type: application/json" \
  -d '{"name": "Updated Name"}'

DELETE /api/projects/:id​

Remove project.

Parameters:

  • id (path): Project ID (numeric)

Response: 204 No Content

Example:

curl -X DELETE http://localhost:3001/api/projects/1

GET /api/projects/:id/tests​

Get tests for a project.

Parameters:

  • id (path): Project ID (numeric)

Response: Test[]

Example:

curl http://localhost:3001/api/projects/1/tests

GET /api/projects/:id/ports​

Get project ports.

Parameters:

  • id (path): Project ID (numeric)

Response: ProjectPort[]

Example:

curl http://localhost:3001/api/projects/1/ports

POST /api/projects/:id/scan​

Scan project for tests.

Parameters:

  • id (path): Project ID (numeric)

Response:

{
  "project": Project,
  "testsFound": number,
  "tests": Test[]
}

Example:

curl -X POST http://localhost:3001/api/projects/1/scan

POST /api/projects/scan/all​

Scan all projects for tests.

Response:

Array<{
  "project": Project,
  "testsFound": number,
  "tests": Test[]
}>

Example:

curl -X POST http://localhost:3001/api/projects/scan/all

Settings​

GET /api/settings​

Get all settings.

Response: Record<string, string>

Example:

curl http://localhost:3001/api/settings

PUT /api/settings/:key​

Update a setting.

Parameters:

  • key (path): Setting key

Request Body:

{
  "value": "setting value"
}

Response:

{
  "key": "setting_key",
  "value": "setting value"
}

Example:

curl -X PUT http://localhost:3001/api/settings/my_setting \
  -H "Content-Type: application/json" \
  -d '{"value": "my value"}'

Error Handling​

All endpoints return appropriate HTTP status codes:

  • 200 - Success
  • 201 - Created
  • 204 - No Content
  • 400 - Bad Request
  • 404 - Not Found
  • 409 - Conflict
  • 500 - Internal Server Error

Error responses include an error field:

{
  "error": "Error message here"
}
